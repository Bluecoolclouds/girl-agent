# Для разработчиков

Обзор архитектуры, исходников и интеграционных точек girl-agent

Этот трек — про **код**. Если ты хочешь:

- понять, как girl-agent устроен внутри,
- встроиться в него (свой LLM-провайдер, свой Telegram-адаптер, свой аддон),
- использовать REST API или headless-протокол из своего приложения,
- внести изменения в репозиторий,

— ты здесь.

## [Стек на одной строке](https://docs.girl-agent.com/docs/developers/\#%D1%81%D1%82%D0%B5%D0%BA-%D0%BD%D0%B0-%D0%BE%D0%B4%D0%BD%D0%BE%D0%B9-%D1%81%D1%82%D1%80%D0%BE%D0%BA%D0%B5)

TypeScript (ESM, Node ≥18.18), `tsup` сборка, `tsx` dev, Node built-in `http` для WebUI, `ws` для WebSocket, `grammy` для bot-режима, `telegram` (gramJS) для userbot, OpenAI + Anthropic SDK для LLM, `@modelcontextprotocol/sdk` для MCP, файловое хранилище в Markdown + JSON. Никаких баз данных, никаких сервисов в зависимостях.

## [Конвенции](https://docs.girl-agent.com/docs/developers/\#%D0%BA%D0%BE%D0%BD%D0%B2%D0%B5%D0%BD%D1%86%D0%B8%D0%B8)

- **ESM-only**: все импорты с расширением `.js` (даже из `.ts`-файлов).
- **Сообщения и комментарии в коде — на русском.** Имена идентификаторов — английские.
- **`process.exit` только из CLI-точек**, runtime никогда не убивает процесс.
- **Никаких глобальных кешей** между профилями — каждый профиль изолирован папкой и `Runtime`-инстансом.
- **Markdown как структура хранения, а не как UI.** Файлы профиля читает и LLM, и человек.
